③ 인프라 변경 (Terraform)

☁️ Terraform Cloud 바로가기

Terraform으로 구성한 AWS 인프라를 다시 Terraform을 통해 얼마나 쉽고 빠르게 삭제할 수 있는지 직접 보여드리겠습니다.

지금까지 설명드린 내용을 바탕으로, 저희가 구축한 세 가지 핵심 결과물을 직접 보여드리겠습니다.

(첫 번째 링크를 클릭하며)
먼저 Streamlit으로 만든 관리자 페이지입니다. 보시는 것처럼, 복잡한 프론트엔드 개발 없이도 데이터 관리와 기능 테스트에 필요한 내부 도구를 Python만으로 빠르고 직관적으로 만들 수 있었습니다.

(두 번째 링크를 클릭하며)
다음은 FastAPI가 자동으로 생성해준 API 문서입니다. 저희는 Scalar UI를 적용하여 더 깔끔한 문서를 만들었습니다. 여기서 실제 API를 직접 테스트해볼 수 있어 프론트엔드 개발자와의 협업에 정말 큰 도움이 되었습니다.

(세 번째 링크를 클릭하며)
마지막으로 Terraform Cloud 워크스페이스입니다. GitHub에 코드를 푸시하면, 바로 이곳에서 Plan과 Apply가 실행되며 모든 인프라 변경 이력이 기록됩니다. 이를 통해 모든 변경 사항을 추적하고 안정적으로 관리할 수 있습니다.

(시연 종료 후, 슬라이드로 복귀)
시연은 여기까지입니다. 지금까지 저희 '멍탐정' 프로젝트의 아키텍처 발표를 들어주셔서 감사합니다. 이제 질문이 있으시면 편하게 해주시기 바랍니다.
