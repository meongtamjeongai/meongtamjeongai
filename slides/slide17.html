프론트엔드: 왜 Streamlit인가? 📊

저희는 사용자 데이터 관리, AI 기능 테스트 등을 위한 관리자 페이지가 필요했습니다.
하지만 프론트엔드 개발에 많은 시간을 쏟을 수는 없었습니다. 저희의 선택은 Streamlit이었습니다.

  • 오직 Python만으로 웹 UI 구현: HTML, CSS, JavaScript를 전혀 몰라도, 익숙한 Python 코드만으로 인터랙티브한 웹 페이지를 만들 수 있습니다.
  • 압도적인 개발 속도: 복잡한 상태 관리나 콜백 없이, 스크립트처럼 위에서 아래로 코드를 작성하면 그대로 웹 화면이 완성되어 개발 시간이 획기적으로 단축됩니다.
  • 데이터 시각화에 최적화: 몇 줄의 코드만으로 데이터프레임, 차트, 지도 등을 손쉽게 시각화할 수 있어 데이터 기반의 관리 도구에 안성맞춤입니다.
이제 프론트엔드, 그중에서도 저희가 관리자 페이지를 만든 경험에 대해 말씀드리겠습니다.

저희는 사용자 데이터를 관리하고, AI 기능을 직접 테스트해볼 수 있는 내부용 관리자 페이지가 필요했습니다. 하지만 한정된 시간 안에 복잡한 프론트엔드 기술을 배우고 개발하는 것은 부담이었습니다.

이때 저희의 눈에 들어온 것이 바로 Streamlit입니다.

Streamlit의 가장 큰 장점은, HTML, CSS, JavaScript를 전혀 사용하지 않고 오직 Python 코드만으로 웹 페이지를 만들 수 있다는 점입니다. `st.button`, `st.text_input` 같은 간단한 함수를 호출하면, 오른쪽 화면처럼 버튼과 입력창이 뚝딱 만들어집니다.

덕분에 저희는 프론트엔드 개발에 드는 시간을 획기적으로 줄이고, 그 에너지를 백엔드와 인프라의 핵심 로직을 개발하는 데 온전히 쏟을 수 있었습니다.
