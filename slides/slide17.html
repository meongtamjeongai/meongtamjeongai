프론트엔드: 왜 Streamlit인가? 📊

관리자 페이지가 필요했지만, 프론트엔드 개발 리소스는 부족했습니다.
이 문제에 대한 저희의 해답은 Streamlit이었습니다.

  • 🐍 순수 Python으로 UI 구현 
    # 예시: 텍스트 입력창 생성
name = st.text_input("이름을 입력하세요")
  • 🚀 압도적으로 빠른 개발 속도 
    # 예시: 버튼 클릭 시 메시지 표시
if st.button("인사하기"):
    st.write(f"{name}님, 환영합니다!")
  • 📈 강력한 데이터 시각화 기능 
    # 예시: 데이터프레임을 테이블로 표시
st.dataframe(my_dataframe)
마지막으로 프론트엔드, 그중에서도 저희가 내부 관리자 페이지를 구축한 경험을 공유해 드리겠습니다.

저희는 사용자 데이터를 조회하거나, 새로 개발한 AI 기능을 직접 테스트해볼 내부 도구가 반드시 필요했습니다. 하지만 한정된 시간 안에 React나 Vue 같은 본격적인 프론트엔드 기술을 사용해 개발하는 것은 현실적으로 불가능에 가까웠습니다. 백엔드 개발자가 프론트엔드까지 빠르게 만들어야 하는 상황이었습니다.

이때 저희의 선택이 바로 Streamlit이었습니다.

첫 번째 장점은, 보시는 것처럼 순수 Python 코드만으로 웹 UI를 구현할 수 있다는 점입니다. 슬라이드의 첫 번째 예시처럼, st.text_input이라는 함수를 호출하기만 하면 웹 화면에 텍스트 입력창이 하나 만들어지고, 사용자가 입력한 값은 `name`이라는 변수에 저장됩니다. 웹 기술을 전혀 몰라도 됩니다.

두 번째는 '압도적으로 빠른 개발 속도'입니다. 두 번째 예시를 봐주세요. Python의 평범한 if문입니다. st.button 함수는 버튼을 만들면서 동시에 '사용자가 버튼을 클릭했는지' 여부를 참/거짓으로 반환합니다. 그래서 보시는 것처럼, '인사하기' 버튼을 누르면 그 아래의 st.write 코드가 실행되어 화면에 환영 메시지를 뿌려주는 로직이 단 두 줄로 완성됩니다. 복잡한 콜백이나 상태 관리가 전혀 필요 없습니다.

마지막으로, '강력한 데이터 시각화 기능'입니다. Streamlit은 데이터 과학을 위한 도구에서 출발했기 때문에 데이터 표현에 매우 강합니다. 세 번째 예시처럼, Pandas 라이브러리로 처리한 `my_dataframe`이라는 데이터 뭉치를 st.dataframe 함수에 그냥 넘겨주기만 하면, 오른쪽 화면처럼 정렬과 검색까지 가능한 멋진 표를 즉시 만들어줍니다.

결론적으로, Streamlit 덕분에 저희는 프론트엔드 개발에 드는 시간을 90% 이상 단축하고, 그 에너지를 백엔드 핵심 로직과 인프라 안정화에 온전히 쏟을 수 있었습니다.
