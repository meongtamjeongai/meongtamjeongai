Q&A 보충: 왜 Route 53 대신 Cloudflare DNS를 선택했는가?

AWS 생태계임에도 불구하고, 저희는 DNS 서비스로 Route 53이 아닌 Cloudflare를 선택했습니다.
이는 비용, 성능, 그리고 부가 기능 측면에서 저희 프로젝트에 더 큰 이점을 제공했기 때문입니다.

구분 ✅ Cloudflare (채택) AWS Route 53 (대안)
💰 비용 기본 DNS 및 DDoS 방어 무료.
소규모 프로젝트에 비용 부담 없음. 		호스팅 영역 및 쿼리 당 비용 발생.
작은 트래픽에도 비용이 청구됨.
🚀 성능 전 세계적으로 가장 빠른 DNS 응답 속도를 제공하는 CDN 네트워크 보유. 안정적이지만, 순수 DNS 속도는 Cloudflare 대비 열세일 수 있음.
🛡️ 보안 및 부가 기능
  • 강력한 무료 DDoS 방어 및 WAF.
  • 프록시(주황색 구름)로 실제 서버 IP 숨김.
  • Cloudflare Access로 코드 없이 제로 트러스트 보안 구현.
  • AWS Shield, WAF 등 별도 서비스 연동 필요. (추가 비용 및 설정 복잡도 증가)

결론: Route 53의 AWS 서비스 간 통합 이점도 분명하지만, 저희 프로젝트 규모에서는 Cloudflare가 제공하는 무료 정책, 압도적인 성능, 그리고 클릭 몇 번으로 구현되는 강력한 추가 보안 기능의 가치가 더 크다고 판단했습니다.

네, '왜 AWS의 Route 53 대신 Cloudflare를 사용했는가?'는 아주 좋은 질문입니다. 저희의 결정 과정을 설명드리겠습니다.

물론 모든 인프라를 AWS로 통일했다면 Route 53이 더 매끄럽게 연동되는 부분도 있습니다. 예를 들어, 인증서 발급 시 DNS 레코드 자동 등록 같은 부분은 더 편했을 겁니다.

하지만 저희는 세 가지 측면에서 Cloudflare가 더 큰 이점을 제공한다고 판단했습니다.

첫째, 비용입니다. 소규모 프로젝트에서 Route 53은 쿼리당 비용이 계속 발생하지만, Cloudflare의 핵심적인 DNS와 DDoS 방어 기능은 무료입니다. 이는 비용에 민감한 프로젝트에 매우 큰 장점입니다.

둘째, 성능입니다. Cloudflare는 전 세계적으로 가장 빠른 DNS 응답 속도를 자랑하는 강력한 CDN 네트워크를 기반으로 동작합니다. 이는 곧 사용자가 저희 서비스에 더 빨리 접속할 수 있음을 의미합니다.

마지막으로, 가장 결정적인 이유였던 보안 및 부가 기능입니다. Cloudflare는 '주황색 구름'으로 표현되는 프록시 기능을 켜는 것만으로 저희의 실제 서버 IP 주소를 숨겨주고, 기본적인 DDoS 공격을 막아줍니다. 특히, 앞서 시연에서 보여드린 'Cloudflare Access' 기능은, 저희가 관리자 페이지를 위한 별도의 로그인 코드를 한 줄도 작성하지 않고도 강력한 접근 제어를 구현할 수 있게 해준 핵심적인 기능이었습니다.

이러한 이유로, 저희는 AWS 서비스와의 네이티브한 통합이라는 이점보다는, Cloudflare가 제공하는 비용, 성능, 그리고 강력한 부가 기능의 가치가 저희 프로젝트에 더 적합하다고 판단하여 최종 선택하게 되었습니다.
