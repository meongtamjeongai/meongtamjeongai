이제 프로젝트의 심장이라 할 수 있는 백엔드에 대해 말씀드리겠습니다. 저희는 수많은 웹 백엔드 프레임워크 중에서 FastAPI를 선택했습니다. 그 이유를 세 가지 핵심으로 요약해서 설명해 드리겠습니다.

첫째, Python의 강력한 생태계는 그대로 쓰면서 '비동기 처리'의 이점을 누릴 수 있다는 점입니다. 저희 프로젝트는 외부 AI API를 호출하거나 데이터베이스에서 데이터를 읽어오는 등, 네트워크 I/O 작업, 즉 기다리는 시간이 많은 작업이 대부분입니다. 전통적인 동기 방식에서는 이런 작업이 끝날 때까지 서버가 다른 모든 요청을 멈추고 기다려야만 합니다. 하지만 FastAPI는 async/await 문법을 통해, API 호출을 보내놓고 응답을 기다리는 시간에 다른 요청을 동시에 처리할 수 있습니다. 이는 서버 자원을 훨씬 효율적으로 사용하게 만들어, 더 적은 수의 서버 인스턴스로도 더 많은 사용자를 감당할 수 있다는 의미입니다.

두 번째 이유는 단기 프로젝트의 성패를 좌우하는 '압도적인 개발 생산성'입니다. FastAPI는 Pydantic이라는 데이터 검증 라이브러리를 기반으로 합니다. 개발자는 그저 Python의 표준 타입 힌트를 사용해 데이터 모델을 정의하기만 하면 됩니다. 예를 들어, API 요청문에 email: str, age: int 라고만 코드에 적어두면, FastAPI가 알아서 클라이언트가 보낸 데이터가 올바른 이메일 형식인지, 나이가 숫자인지 등을 자동으로 검증하고 에러 메시지까지 생성해줍니다. 여기서 끝이 아닙니다. 이 코드로부터 프론트엔드 개발자가 즉시 테스트해볼 수 있는 대화형 API 문서(Swagger UI나 Scalar)를 자동으로 만들어줍니다. API 명세를 따로 작성하고 공유하는 데 드는 시간을 완벽하게 없애준 셈입니다. 버그는 줄고, 소통 비용은 감소하니 개발 속도가 빨라질 수밖에 없었습니다.

마지막으로, '가볍고 현대적인 설계' 사상입니다. FastAPI는 Flask처럼 마이크로 프레임워크 철학을 공유해 매우 가볍고, 핵심 기능만 제공하여 배우는 데 걸리는 시간(러닝커브)이 매우 짧습니다. 그러면서도 대규모 애플리케이션에 필수적인 의존성 주입(Dependency Injection, DI) 시스템을 프레임워크 차원에서 내장하고 있습니다. 이게 왜 중요하냐면, 데이터베이스 연결 세션이나, 인증 로직, 설정 값처럼 여러 기능에서 공통적으로 사용되는 자원들을 매우 깔끔하고 유연하게 관리할 수 있게 해줍니다. 덕분에 코드의 재사용성이 높아지고, 특히 유닛 테스트를 작성하기가 매우 쉬워져 코드의 품질과 유지보수성을 자연스럽게 확보할 수 있었습니다.
